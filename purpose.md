# AI 求职助手 - 项目定位与愿景 (purpose.md)

## 1. 核心愿景与策略
**AI 求职助手**旨在通过自动化职位发现、语义过滤以及求职材料的个性化定制，解决现代求职过程中的繁琐流程。系统采用 **API 优先**与**零信任安全**架构，完全使用公开、免认证或有官方支持的 JSON 接口获取数据，从而完美避开各类反爬虫机制与 IP 限流风险。

## 2. 核心功能支柱

### A. 语义职位匹配引擎 (Matchmaker / 智能匹配)
* **目标：** 过滤低相关性噪音，精准定位高价值岗位。
* **核心业务流程（增量演进）：**
  `Streamlit 触发上传简历` -> `Gemini 动态提取关键词` -> `人工前端确认 (Human-in-the-loop)` -> `驱动精确检索`。
* **机制：** 抓取非结构化的原始招聘数据，利用自然语言处理 (NLP) 和向量嵌入技术解析职位描述 (JD)，并与候选人的技能画像、目标地点及个人偏好进行匹配度评分。
* 动态检索与人工确认（数据节点 1）： 在日内瓦、瑞士及远程工作（Remote）场景下，系统拒绝硬编码固定的检索范围。系统首先通过大模型从候选人上传的简历中动态提取 5~10 个最能体现其当前核心竞争力的多维关键词。在进入后台 API 检索前，必须经过前端用户界面（UI）进行人工增删、修改及勾选确认（Human-in-the-loop）。系统设定硬性熔断上限（Hard Cap）：最终触发检索的关键词总数绝对禁止超过 20 个，以严格控制 API 的并发调用风险与 Token 消耗成本。

### B. 简历/求职信自动定制系统 (Automated Tailoring & Cover Letter)
* **目标：** 一键生成与职位高度契合的个性化申请材料。
* **机制：** 动态交叉比对候选人的简历元数据与匹配职位的具体 JD，智能生成定制化的求职信 (Cover Letter) 以及简历亮点修改建议。
* **安全性：** 确保候选人敏感数据不被外泄，所有核心解析逻辑均在本地或通过高安全性的 API 加密传输。
